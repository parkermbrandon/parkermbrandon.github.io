Build and sign

Make sure the AIO Control Panel is built.

The library_build.sh and app_build.sh scripts will build a Release version of the library & app and a companion dSYM symbols file, and sign it using the Developer ID certificate present in the keychain.

Run from the root project directory:

sh ./MacScripts/library_build.sh
sh ./MacScripts/app_build.sh

Create a release DMG

Install the create-dmg utility using homebrew: brew install create-dmg

Or from GitHub: https://github.com/andreyvit/create-dmg

Once the app has been built, use create_dmg.sh to package it in a DMG:

./MacScripts/create_dmg.sh "Echo AIO x.y.z" EchoAIO_x.y.z.dmg

The first argument is the volume name when the DMG is mounted.

The second argument is the DMG file that is created.

Notarize the DMG

If this is the first time you're notarizing a release, run notarize_setup.sh to add the appropriate account information to the keychain.

Use notarize_dmg.sh to notarize the dmg built in the previous step.

sh ~/MacBuildScripts/notarize_dmg.sh com.echoaudio.EchoAIO EchoAIO_x.y.z.dmg

The first argument is the bundle identifier for the app.

The second argument is the dmg file created by create_dmg.sh.

If notarize_dmg.sh succeeds, the resulting DMG file is ready for distribution.
